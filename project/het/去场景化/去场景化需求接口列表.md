## 去场景化需求接口列表

### 规则配置

WEB:

```
1. CRUD规则主信息
2. CRUD规则条件
3. CRUD规则动作
4. 角色管理

```

APP/服务

```
1. 查询规则模板(列表)
2. 匹配生成用户规则
3. 刷新(重新匹配生成)用户规则
4. 自定义用户规则(自定义规则主信息，条件和动作)
```

### 规则执行

服务调用：

```
1. 开启用户规则
2. 关闭用户规则
3. 重启用户规则
4. 规则注册。执行

```

C端接口：

```
1. 开启用户规则
2. 关闭用户规则
3. 重启用户规则

```

### 总计

大概22个接口，每个接口2小时，28 * 2 = 56 个小时，需要1.5周开发时间
